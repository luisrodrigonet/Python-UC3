## Desafio - Ajuste do Modelo Cliente para Perfil Integrado ao Usuário Django

### Objetivo

Nesta atividade, você vai modificar a classe **Cliente** para que funcione como um perfil estendido vinculado ao modelo padrão de usuário do Django. Além disso, o modelo Cliente será enriquecido com campos que armazenam informações pessoais importantes para o cadastro do cliente em uma loja de e-commerce.

### Descrição da tarefa

1. Implemente na classe **Cliente** um campo `OneToOneField` que relacione o cliente ao modelo `User` do Django, garantindo que cada usuário autenticado possua exatamente um perfil cliente associado.

2. Mantenha o campo `data_cadastro` e adicione os seguintes novos campos ao modelo Cliente:
   - `foto` (imagem de perfil, opcional)
   - `sexo` (campo restrito a opções: Masculino, Feminino, Outro, Prefiro não dizer)
   - `nome_social` (nome social do cliente, opcional)
   - `data_nascimento` (data de nascimento, opcional)
   - `cpf` (campo texto para CPF com validação básica e obrigatório ser único)

3. Remova da classe Cliente os campos que duplicam informações já presentes no modelo User, como nome completo e e-mail, passando a usar o modelo User para essas informações.

4. Garanta que o método `__str__` da classe Cliente retorne o nome social (se preenchido) ou o nome completo do usuário (`user.get_full_name()`).


### Pontos importantes para implementação

- Importe o modelo **User** com:
  ```python
  from django.contrib.auth.models import User
  ```

- Use `OneToOneField` para associar o usuário ao cliente:
  ```python
  user = models.OneToOneField(User, on_delete=models.CASCADE, related_name='cliente_perfil')
  ```

- Para o campo `foto`, utilize:
  ```python
  foto = models.ImageField(upload_to='fotos_perfil/', blank=True, null=True)
  ```

- Defina uma lista com as opções de sexo para o campo `sexo`.

- Defina `cpf` como `CharField` com `unique=True` e um limite máximo de 14 caracteres para armazenar o CPF formatado.

- Deixe campos opcionais (`blank=True`, `null=True`) para não bloquear o cadastro inicial.

### Objetivos de aprendizado

- Entender a integração entre o modelo User do Django e um modelo de perfil personalizado.
- Praticar o uso do `OneToOneField` para extensão de usuários.
- Aprender a adicionar novos campos com validações e configurações específicas.
- Compreender boas práticas para armazenamento de dados pessoais em aplicações web seguras.

### Resultado esperado

Ao final da atividade, o modelo Cliente deverá representar o perfil completo do usuário autenticado, armazenando informações pessoais relevantes para o comércio eletrônico, e estar devidamente vinculado à estrutura de autenticação do Django.
